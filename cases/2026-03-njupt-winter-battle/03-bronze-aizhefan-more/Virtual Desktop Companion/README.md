## 项目概述

AI桌面宠物系统是一个集成了人格模拟、专业知识检索和WSL2命令执行能力的智能桌面应用。系统采用客户端-服务器架构，通过精心设计的LLM工程和RAG技术，实现了智能对话和WSL2命令控制功能。

## 项目结构

### 整体架构

```
├── client/             # 客户端代码
│   ├── app.js          # Vue.js应用主文件
│   ├── index.html      # 前端页面
│   ├── local-server.js # 本地Node.js服务器

│   └── styles.css      # 样式文件
├── server/             # 服务器端代码

│   ├── prompt_assembler.py # 提示词组装器
```

## LLM工程设计

### 提示词工程

提示词工程是系统的核心部分，通过精心设计的提示词模板，引导LLM生成符合预期的响应。系统采用分层设计的提示词结构：

#### 基础系统提示词

基础系统提示词定义了AI的核心身份和行为准则：

- 核心身份：AI桌面宠物，有自己的性格和情绪
- 核心原则：回答简短、真诚、有个性
- 人格参数：简洁度、哲思倾向、顽皮度、温暖度等
- 情绪状态：反映当前的情感状态

#### 模式特定提示词

系统支持多种交互模式，每种模式有特定的提示词模板：

- **普通聊天模式**：适用于日常交流，回答长度控制在20-50字
- **专业模式**：适用于技术问题，直接回答核心问题
- **哲思模式**：适用于深度话题，提供有哲理的回应
- **摆烂模式**：适用于简单问题，用轻松幽默的方式回应
- **恶搞模式**：适用于轻松氛围，使用网络梗和幽默

#### WSL2命令模式

针对WSL2命令执行，系统设计了专门的提示词模板，引导LLM生成正确的命令格式：

- 明确的JSON输出格式
- 命令的详细描述
- 参数的正确处理
- 安全执行的考虑

### RAG系统设计

RAG（Retrieval-Augmented Generation）系统是提升LLM专业知识能力的关键组件：

#### 知识库构建

- **数据来源**：从官方文档和GitHub仓库收集WSL2相关知识
- **知识处理**：将知识按类别和子类别组织，提取关键信息
- **向量存储**：使用ChromaDB构建向量数据库，存储知识嵌入

#### 知识检索

- **语义搜索**：基于用户查询的语义相似度进行检索
- **分类过滤**：根据检测到的知识类别进行过滤
- **置信度评估**：计算检索结果的置信度，确保知识的可靠性

#### 知识集成

- **格式标准化**：将检索到的知识格式化为统一的结构
- **上下文融合**：将知识与对话上下文融合，提供连贯的回答
- **重要性排序**：根据知识的重要性和相关性进行排序

## 服务器端实现

### API设计

服务器提供RESTful API和WebSocket接口：

- **聊天接口**：处理用户消息，返回AI回复
- **记忆接口**：管理会话历史和用户画像
- **技能接口**：提供特定功能的调用

### 核心流程

1. **请求接收**：接收客户端的聊天请求
2. **语义分析**：分析用户意图和情绪
3. **记忆检索**：从记忆系统中检索相关上下文
4. **知识检索**：使用RAG系统检索专业知识
5. **提示词组装**：构建完整的LLM提示词
6. **LLM推理**：调用LLM生成响应
7. **响应处理**：解析LLM输出，生成最终响应
8. **记忆更新**：将对话内容更新到记忆系统

### 多模式支持

系统支持两种主要模式：

- **聊天模式**：使用Ollama API，注重人格模拟和自然对话
- **WSL2模式**：使用DeepSeek API，注重命令准确性和执行效果

## 客户端实现

### 前端设计

- **界面布局**：可拖拽的宠物窗口，包含聊天区域和控制按钮
- **交互方式**：文本输入、模式切换、命令执行
- **视觉效果**：Live2D角色动画，根据情绪状态变化

### 本地服务器

本地服务器负责WSL2命令的执行：

- **命令接收**：接收客户端的命令执行请求
- **命令执行**：在WSL2环境中执行命令
- **结果处理**：捕获命令输出，处理编码问题
- **结果返回**：将执行结果返回给客户端

### 通信流程

1. **用户输入**：用户在前端输入消息
2. **模式判断**：根据当前模式决定处理方式
3. **服务器请求**：向远程服务器发送请求
4. **命令生成**：服务器生成WSL2命令
5. **本地执行**：客户端调用本地服务器执行命令
6. **结果显示**：将命令执行结果显示在前端
